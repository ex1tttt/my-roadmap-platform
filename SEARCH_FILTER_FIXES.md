# Исправления ошибок поиска и фильтрации

## Проблемы найдены и исправлены ✅

### 1. **Фильтр по категориям не работал** ❌→✅
**Причина:** Несоответствие ID категорий!
- В БД и constants используются ID: `"programming"`, `"cooking"`, `"marketing"` и т.д.
- Но фильтр на главной странице искал по русским названиям: `"Программирование"`, `"Готовка"`
- Результат: запрос находил 0 карточек

**Исправление:**
- Импортирован `categories` из constants
- Фильтр теперь динамически генерируется из `categories.id`
- Показываются названия на текущем языке пользователя

**Файл:** `app/page.tsx`

### 2. **Сортировка по популярности была неправильной** ❌→⚠️
**Проблема:** Локальная сортировка применялась только к 16 карточкам на странице
- Результат: разные страницы показывали разные порядки

**Временное решение:**
- Удалена неправильная локальная сортировка по `averageRating`
- Добавлена правильная сортировка по `likesCount` ПОСЛЕ загрузки данных

**Оптимальное решение (требует миграции):**
- Нужно создать в БД столбец `likes_count` с триггерами для автоматического обновления
- See: `db/migration_likes_count.sql`

### 3. **Отсутствует денормализация количества лайков** ⚠️
**Проблема:** Нет быстрого способа отсортировать карточки по популярности при пагинации

**Решение:** Нужно выполнить миграцию в Supabase

## Как применить миграцию likes_count

### Шаг 1: Откройте Supabase SQL Editor
1. Перейти на https://supabase.com/dashboard
2. Выбрать проект
3. Перейти в SQL Editor

### Шаг 2: Скопируйте и выполните миграцию
Скопируйте содержимое `db/migration_likes_count.sql` и выполните в SQL Editor.

Это добавит:
- ✅ Столбец `likes_count` в таблице `cards`
- ✅ Триггер при добавлении лайка (инкремент)
- ✅ Триггер при удалении лайка (декремент)
- ✅ Пересчет существующих лайков

### Шаг 3: После миграции
После выполнения миграции, сортировка по популярности будет работать правильно при пагинации.

## Тестирование

### Фильтр по категориям
✅ Выберите категорию на главной странице - результаты должны фильтроваться правильно
